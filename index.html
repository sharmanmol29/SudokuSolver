1) What is Backtracking?

Backtracking is a depth-first search method that tries a choice, moves forward if valid, and returns to undo that choice when it reaches a dead end. Think of solving a maze: you follow one path, and if it is blocked, you walk back to the last junction and try a new route.

2) How the Sudoku Solver Works

The solver scans for an empty cell, tests numbers 1 through 9, and places one only if it does not violate row, column, or 3x3 box constraints. It then recurses to solve the next empty cell.

solve(board):
  empty = findEmpty(board)
  if not empty: return true
  row, col = empty
  for num in 1..9:
    if isValid(board, row, col, num):
      board[row][col] = num
      if solve(board): return true
      board[row][col] = 0
  return false

3) Why This is a DAA Project

Sudoku is a classic constraint satisfaction problem where each assignment must satisfy strict rules. Generalized Sudoku belongs to NP-complete classes, and this project demonstrates algorithm design through recursion, pruning invalid branches early, and systematic search.

4) Complexity Analysis

Time: O(9^m), where m is the number of empty cells, since each cell can branch up to 9 possibilities in the worst case.

Space: O(m), due to recursion stack depth.

Best case: heavily constrained puzzle with aggressive pruning.
Average case: moderate branching with frequent dead-end cuts.
Worst case: many empties with late conflicts requiring deep exploration.

Algorithm: Recursive Backtracking with constraint validation and unique-solution puzzle generation.
